Proses Desain Database

Perencanaan

  1. Mendefinisikan Tujuan dan Kegunaan Database
    Langkah dalam proses perancangan basis data yang pertama adalah menentukan arah dan tujuan basis data. Arah dan tujuan database membantu perancang database untuk menentukan tujuannya dengan jelas. Bergantung pada ruang lingkup proyek, perancang database dapat menggambarkan detailnya. Misalnya, untuk desain database yang lebih kecil, cakupannya dapat dibatasi untuk merekam informasi bisnis tertentu. Namun demikian, maksud dan tujuan database untuk aplikasi tingkat perusahaan dapat berupa dokumen detail yang dapat berfungsi sebagai dokumen penting bagi tim pengembangan database.

  2. Menentukan Persyaratan Data Pengguna
    Spesifikasi kebutuhan pengguna adalah bagian terpenting dari dokumen yang menentukan ruang lingkup proyek perangkat lunak. Dari perspektif desain database, URS juga mendefinisikan ruang lingkup database dan kebutuhan data pengguna yang harus didukung oleh aplikasi perangkat lunak. URS juga menyediakan input desain penting bagi tim desain database untuk menggabungkan dan mengembangkan database yang mendukung kebutuhan pengguna dalam hal data, keamanan, dan fungsionalitas.

  3. Membuat Data Model
    Model data adalah alat sederhana yang digunakan oleh desainer database untuk memvisualisasikan dan mengkomunikasikan fitur desain database pada berbagai tahapan proses desain dan pengembangan database. Model data ini kemudian digunakan sebagai cetak biru oleh desainer database pada berbagai tahap proyek desain dan pengembangan database. Model database membuatnya sangat mudah untuk memvisualisasikan dan mengkomunikasikan fitur desain database yang berbeda dengan bantuan diagram.

  4. Membangun Model Data Konseptual
    Model data konseptual merupakan diagram sederhana yang hanya menyoroti berbagai entitas database yang ada dan hubungan antar entitas tersebut. Model data konseptual adalah yang pertama dibuat untuk menangkap kebutuhan data pengguna selama tahap awal proses desain database.
    Entity Relationship Diagram (ERD) adalah contoh model data konseptual yang dibangun selama proses desain database. Basis data konseptual menggambarkan tampilan basis data pada tingkat pengguna akhir dengan diagram sederhana yang tidak bergantung pada DBMS.

  5. Normalisasi Database
    Tujuan utama dari normalisasi database adalah untuk meminimalkan adanya data duplikat dalam tabel. Duplikasi data adalah akar penyebab anomali database. Anomali database dapat menyebabkan keadaan database tidak konsisten yang menyebabkan hasil yang tidak akurat dari berbagai operasi database.

  6. Mendefinisikan Spesifikasi Kebutuhan Database
    Bergantung pada skala proyek dan kompleksitas desain database, tim proyek perlu menyelesaikan dan menentukan fitur desain dan spesifikasi database yang diusulkan. Basis data setelah pengembangan harus mendukung persyaratan data pengguna dan fungsionalitas perangkat lunak sesuai dengan dokumen USR.

Pengembangan

  1. Memilih DBMS Untuk Membangun Database
    Langkah pertama dalam tahap pengembangan database adalah memilih Database Management System (DBMS) yang sesuai. Pemilihan DBMS tergantung pada jenis aplikasi perangkat lunak, arsitektur sistem basis data, dan fungsionalitas basis data.

  2. Membangun Model Data Fisik
    Setelah memilih DBMS yang akan digunakan untuk mengembangkan database, langkah selanjutnya dalam pengembangan database adalah membuat model data fisik. Model data fisik dibuat dengan memperluas model data logis dengan beberapa detail tambahan yang spesifik DBMS. Model data fisik bergantung pada DBMS karena berisi detail yang dapat diterapkan pada DBMS tertentu.

  3. Pengembangan Database Dengan DBMS
    Pada tahap ini, semua detail desain dan cetak biru sekarang siap untuk benar-benar membangun database. Beberapa Sistem Manajemen Basis Data Relasional (RDBMS) yang paling umum digunakan termasuk MySQL, Oracle, MS SQL Server, Mango DB dan masih banyak lagi DBMS yang tersedia di dunia basis data.
    Pekerjaan pengembangan database dimulai dengan membuat tabel terlebih dahulu. Hubungan tabel ditentukan menggunakan kunci database utama dan asing untuk setiap tabel. Batasan database juga dapat ditentukan pada tahap ini

  4. Pengujian dan Validasi Database
    Pengujian basis data adalah langkah penting yang memastikan akurasi, konsistensi, dan integritas data yang disimpan ke dalam basis data. Setelah membangun database, langkah selanjutnya adalah menguji coba database dengan data sampel dan memvalidasi hasilnya. Basis data setelah pengembangan mengalami pengujian ekstensif dan proses validasi termasuk fungsionalitas dari berbagai operasi basis data.

  5. Dokumentasi Database Untuk Referensi di Masa Mendatang
    Dokumentasi database adalah manual referensi penting dalam setiap siklus hidup pengembangan perangkat lunak (SDLC). Tim manajemen proyek juga membuat dan memelihara dokumentasi yang diperlukan terkait dengan pengembangan database. Dokumentasi ini adalah manual referensi penting bagi tim pengembang untuk memasukkan setiap perubahan baik dalam aplikasi perangkat lunak front end atau ke dalam struktur database atau fungsionalitas tambahan.
